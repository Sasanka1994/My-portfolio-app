This looks like an "app"!!!

What do you mean by "app" exactly? Regardless, this website looks the way it does because it's using Ionic Framework. It's probably overkill for this use case, and I may change it later, but I'm enjoying Ionic+angular recently.

Also, this is running on NGINX in a Docker container, in AWS' Elastic Beanstalk!. Why Docker and AWS Beanstalk? Docker is a tool designed to make it easier to create, deploy and run applications by using containers. Thus containers allow to package up an application with all of the part it needs and deploy it as one package. Furthermore AWS' Beanstalk is an easy-to-use service for deploying and scaling web applications and services developed with Java, .NET, PHP, Node.js, Python, Ruby, Go, and Docker on familiar servers such as Apache, Nginx, Passenger, and IIS. From this I can make a module to learn CI/CD pipelines in hands on experience.

I am planning to deploy my own automated Jenkins-based build porcess, too; Jenkins will be scanning my remote git repository for changes,then if it finds any, builds and deploys the containerised application automatically, versioning it using per-environment logic and a date-time string.
